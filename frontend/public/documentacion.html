🏠 1. Instalación y Ubicación

¿Dónde se instala FabriControl?

FabriControl es una aplicación desktop que se instala directamente en una PC con Windows. Durante la instalación, todos los archivos se copian a:

📁 Ubicación por defecto:
C:\Program Files\FabriControl\

Requisitos del Sistema

Componente Mínimo Recomendado
Sistema Operativo Windows 10 Windows 11
Procesador Intel Core i3 o equivalente Intel Core i5 o superior
Memoria RAM 4 GB 8 GB o más
Disco Duro 10 GB libres 50 GB libres (para archivos)
Red No requerida (modo local) Red LAN para múltiples usuarios

Estructura de Archivos

Después de la instalación, FabriControl crea la siguiente estructura:

C:\Program Files\FabriControl\
├── 📁 frontend/          → Interfaz web (React)
├── 📁 backend/           → Servidor (Python/FastAPI)
├── 📁 mongodb/           → Base de datos (si es local)
├── 📁 uploads/           → Archivos subidos (PDFs, fotos)
├── 📁 backups/           → Respaldos automáticos
└── 📄 FabriControl.exe   → Ejecutable principal

🗄️ 2. Base de Datos MongoDB

¿Qué es MongoDB?

MongoDB es la base de datos donde FabriControl guarda toda tu información de negocio:

  • 📋 Cotizaciones y presupuestos
  • 📦 Inventario de materiales
  • 👥 Datos de clientes y proveedores
  • 🏭 Órdenes de producción
  • 📊 Reportes y estadísticas
💡 Analogía simple:
Piensa en MongoDB como un archivero digital súper organizado donde se guardan todos tus documentos del negocio. Es como tener todas tus carpetas, pero de forma electrónica y mucho más rápida de buscar.

Opciones de MongoDB

Durante la instalación, puedes elegir entre 2 opciones:

Opción ¿Qué significa? ¿Para quién?
🏠 MongoDB Local La base de datos se instala en tu PC servidor. Todos los datos están físicamente en tu computadora. ✅ Talleres pequeños
⚠️ Requiere internet para validar licencia
✅ Máxima privacidad
☁️ MongoDB en la Nube La base de datos está en internet (MongoDB Atlas). Tus datos están en servidores seguros en línea. ✅ Múltiples sedes
✅ Acceso remoto
✅ Backups automáticos

¿Dónde se guardan tus datos?

🏠 Si elegiste MongoDB Local:
Los datos se guardan en: C:\Program Files\FabriControl\mongodb\data\
Tamaño inicial: ~500 MB
Crecimiento: +10-50 MB por mes (depende del uso)
☁️ Si elegiste MongoDB en la Nube:
Los datos están en servidores de MongoDB Atlas (AWS)
Ubicación física: Seleccionable (EE.UU., Europa, Brasil, etc.)
Encriptación: Sí, automática

Tamaño de la Base de Datos

Ejemplo de crecimiento típico:

  • Taller pequeño (50 órdenes/mes): ~2 GB por año
  • Taller mediano (200 órdenes/mes): ~8 GB por año
  • Fábrica grande (1000+ órdenes/mes): ~40 GB por año
⚠️ Importante: Los archivos adjuntos (fotos, PDFs) NO se guardan en MongoDB. Se guardan por separado en la carpeta uploads/ (ver siguiente sección).

💾 5. Respaldos y Seguridad

Backups Automáticos

FabriControl hace respaldos automáticos de tu información:

📦 Respaldos Automáticos:
• Frecuencia: Diario (a las 2:00 AM)
• Ubicación: C:\Program Files\FabriControl\backups\
• Retención: Últimos 7 días
• ¿Qué incluye? Base de datos completa (MongoDB)

¿Cómo hacer un respaldo manual?

Desde el panel de administración:

  1. Ve a Configuración → Respaldos
  2. Haz clic en "Crear Respaldo Ahora"
  3. Espera 1-5 minutos (depende del tamaño de datos)
  4. Descarga el archivo .backup y guárdalo en lugar seguro

¿Cómo restaurar un respaldo?

  1. Ve a Configuración → Respaldos → Restaurar
  2. Selecciona el archivo .backup
  3. Confirma la restauración
  4. El sistema se reiniciará automáticamente
⚠️ Importante:
• Los archivos adjuntos (fotos, PDFs) NO están incluidos en el backup automático de MongoDB
• Debes respaldar manualmente la carpeta uploads/
• Recomendación: Copia la carpeta uploads/ a un disco externo semanalmente

Seguridad de los Datos

  • 🔒 Encriptación: Todos los datos sensibles están encriptados
  • 👤 Control de acceso: Cada usuario tiene su contraseña y permisos
  • 📝 Registro de actividad: Se registra quién hizo qué y cuándo
  • 🔐 Autenticación: Tokens JWT para máxima seguridad

🔐 7. Sistema de Licencias y Activación

¿Cómo funciona el sistema de licencias?

FabriControl utiliza un sistema de licencias centralizado que verifica tu suscripción cada vez que inicias sesión.

⚠️ IMPORTANTE - Conexión a Internet REQUERIDA:
FabriControl SIEMPRE necesita conexión a internet para validar tu licencia al iniciar sesión. Sin internet, no podrás acceder al sistema, incluso si está instalado localmente.

Estados de Licencia Posibles

trial Cliente nuevo en período de prueba (30 días) ✅ Sí (si no expiró)
activa Cliente pagó su mensualidad ✅ Sí
suspendida Cliente no pagó, suspendido temporalmente ❌ No
inactiva Cliente canceló el servicio ❌ No
expirada Trial terminó sin pago ❌ No

Formato de Licencia

📋 Formato estándar:
FABRI-XXXX-XXXX-XXXX

Ejemplo: FABRI-1234-5678-ABCD

Renovación Mensual

Cada mes, antes del día de renovación:

  1. El sistema envía un recordatorio de pago automático
  2. El cliente realiza el pago mensual ($49 o $129)
  3. El soporte verifica el pago recibido
  4. El estado de la licencia se mantiene como "activa"
  5. Si el cliente NO paga, el estado cambia a "suspendida"
⚠️ Sin Pago = Sin Acceso:
Si no se recibe el pago mensual, el acceso al sistema se bloqueará automáticamente el día de renovación. Todos los datos se mantienen seguros y se reactivan al recibir el pago.

Preguntas Frecuentes sobre Licencias

❓ ¿Puedo usar FabriControl sin internet?
NO. FabriControl SIEMPRE requiere conexión a internet para validar tu licencia al iniciar sesión. Sin internet, no podrás acceder al sistema.
❓ ¿Qué pasa con mis datos si no pago?
Tus datos (cotizaciones, órdenes, inventario) están 100% seguros. Se mantienen intactos en tu servidor local o en la nube. Cuando reactives tu licencia, podrás acceder a todo como antes.
❓ ¿Cómo cancelo mi suscripción?
Simplemente deja de pagar y tu licencia pasará a estado "suspendida". Si quieres eliminar definitivamente tu cuenta y datos, contacta a soporte.
❓ ¿Puedo reactivar después de cancelar?
Sí, en cualquier momento. Realiza el pago mensual correspondiente y el soporte reactivará tu licencia. Todos tus datos estarán disponibles.
